# FULIGIN — Action List

## Style Authority Files
Every UI implementation must reference these two sources before touching a pixel:
- **`vibe_presentation.html`** / **`FULIGIN_Guide.pdf`** — canonical visual mockups. Pages 2 and 5 of the PDF show the in-game Qud/Noctis style; page 4 shows the title screen.
- **`FULIGIN_HUD_DESIGN_BRIEF.md`** — detailed spec for FF7R panel geometry, bar sizing, glow values, and color hex codes.

## The Two-Style Rule

**In-game HUD, inventory, item cards, tooltips, Cugel-9 logs, resource readouts** → **Caves of Qud / Noctis terminal aesthetic**: monospace (`Share Tech Mono`), bracket-corner rectangular panels, `[LABEL: VALUE]` notation, acid green data values, cyan structure lines, magenta category tags, all-caps.

**Main menu, pause menu, settings, reliquary selection, shop, warp screen** → **FF7 Remake menu aesthetic**: angled parallelogram panels with ~20° diagonal corner cuts, thin 1px URTH CYAN glowing borders, segmented gauge bars, chevron `>` active-state markers, dark near-black glassmorphism fill.

**34. Star-Shadow Radiation Shielding** ✅ [DONE] — todo.md Lore §3. Extends Item 33: during an ACTIVE flare, a per-frame asteroid-shadow scan in `update_progression()` walks the asteroid pool and checks whether any large/medium asteroid (size ≥ 2) lies between the player and the dying sun (origin). Algorithm: compute `u = -player_pos / |player_pos|` (unit vector toward origin); for each asteroid `A`, reject if its distance from origin ≥ player's; project `(A - player) · u`; reject if `t < 0` or `t > |player_pos|`; compute perpendicular distance; accept if `perp ≤ SOLAR_FLARE_SHADOW_HALF_W + 0.6 × asteroid.radius`. If any asteroid passes, `solar_flare_in_shadow = 1`. The drain hook in `update_player_physics()` substitutes `SOLAR_FLARE_SHADOW_MULT = 1.4f` for `SOLAR_FLARE_FUEL_MULT = 3.0f` when shielded — taking cover under a big rock more than halves the flare's reactor drain. Visual: `render_entities()` draws a low-alpha cinnabar vector tether from each shielding asteroid's player-facing edge out to the player; alpha = `60 + 140 × centre_factor × pulse(8rad/s)` so the line shimmers during the flare and brightens as the player centres in the corridor. Drawn before asteroids so silhouettes overlay the lines. Constants: `SOLAR_FLARE_SHADOW_MULT = 1.4f`, `SOLAR_FLARE_SHADOW_HALF_W = 80.0f` (world units). Verified clean build, 327KB exe, no new warnings.
